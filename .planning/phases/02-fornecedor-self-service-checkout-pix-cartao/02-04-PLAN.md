## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anon-browser → SSE Route Handler | Session check first; tenant-org match second; only then open stream |
| Postgres LISTEN channel ↔ many web instances | Fan-out is automatic — multiple instances LISTENing on same channel all receive NOTIFY (Pitfall 12) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-04-01 | Spoofing | SSE consumer impersonates another tenant's event | mitigate | session.activeOrganizationId === tenant.id check BEFORE stream open |
| T-02-04-02 | Information disclosure | NOTIFY payload contains PII | mitigate | Pitfall 3 + Pattern 3 — IDs only; no PII in pg_notify payload |
| T-02-04-03 | DoS | SSE connection leak (Pitfall 4) | mitigate | req.signal.abort wired to clearInterval + conn.end() + controller.close(); MAX_SSE_CONN cap |
| T-02-04-04 | DoS | NOTIFY storm during high reservation rate | accept | Postgres handles thousands of NOTIFYs/sec; piloto Trindade volume is modest; observability via Pino bindings |
| T-02-04-05 | Tampering | NOTIFY payload exceeds 8000 bytes | mitigate | Pitfall 3 — Zod validation in lot-notify-channel keeps payload short; emitOutboxEventAndNotify only ships IDs |
| T-02-04-06 | Spoofing | Forged NOTIFY from another DB-level role | mitigate | LISTEN is global per role; only authenticated app roles can NOTIFY; gitleaks + env management prevents leak of DATABASE_URL |
| T-02-04-SC | n/a | No new npm packages | n/a | postgres.js already installed |
